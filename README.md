# MOF Hydrogen Screening Handoff

Bienvenido P. Montoya III's MOF hydrogen storage screening project. The repository packages the machine-learning notebooks, the 12,986-MOF descriptor/ranking table, reproducibility split indices, and curated RASPA/GCMC inputs and outputs requested for journal handoff.

## Repository Layout

- `notebooks/` - six ML notebooks, patched to use repository-relative paths.
- `data/descriptors/` - 12,986-MOF descriptor table and recovered RF ranking source.
- `data/raw/` - local raw/labeled training CSVs copied from the recovered materials.
- `data/processed/` - canonical PS/TPS/REAL tables used as notebook fallbacks.
- `models/` - target location for saved `.joblib` models generated by notebooks.
- `splits/` - precomputed train/test split indices and JSON manifests.
- `raspa/` - RASPA templates, force fields, hydrogen definitions, validation CIFs, run inputs, and curated GCMC outputs.
- `figures/` and `outputs/` - publication figures and analysis outputs.
- `manifests/` - inventory and handoff checklist files with hashes and provenance notes.

## Key Handoff Assets

- Descriptor CSV for 12,986 MOFs: `data/descriptors/mof_descriptors_12986.csv`
- RF ranking source for 12,986 MOFs: `data/descriptors/final_combined_ranking_rf_12986.csv`
- Split indices: `splits/`
- RASPA manifest: `raspa/manifests/raspa_file_manifest.csv`
- Data manifest: `manifests/data_manifest.json`
- Model status manifest: `models/model_manifest.json`

## Environment

Create the Python environment with:

```bash
conda env create -f environment.yml
conda activate mof-h2-screening
python -m ipykernel install --user --name mof-h2-screening
```

`requirements.txt` is also provided for pip-based environments. RASPA itself is not installed by this environment file; set `RASPA_DIR` to the local RASPA simulations directory before running GCMC cases.

## Running the ML Notebooks

Open notebooks from the repository root or from `notebooks/`. Each notebook detects the project root automatically.

Run order inside a notebook:

1. Run Cell A to regenerate cleaned canonical CSVs in `outputs/ml/<algorithm>/`.
2. Run Cell B to train or load models, save metrics, save `.joblib` files under `models/<algorithm>/`, and write split manifests under `splits/<algorithm>/`.

By default `RETRAIN_MODELS = False`. If a matching saved best/final model pair already exists, the notebook loads it. Set `RETRAIN_MODELS = True` to force regeneration.

## RASPA Materials

The curated RASPA handoff is under `raspa/`:

- `raspa/templates/` has reusable `simulation.input` and `run.sh`.
- `raspa/forcefields/` includes `Generic` and `Generic_H2fix`.
- `raspa/molecules/` includes the hydrogen molecule definitions used by the simulation inputs.
- `raspa/validation_cifs/` contains the six validation CIFs.
- `raspa/runs/<MOF>/<case>/` contains the packaged simulation inputs, run scripts, stdout, and raw `.data` output files.

To rerun a packaged case:

```bash
cd raspa/runs/VAGMAT/PS_low
export RASPA_DIR="$HOME/RASPA/simulations"
bash run.sh
```
